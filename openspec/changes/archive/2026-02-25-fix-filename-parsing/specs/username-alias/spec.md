## MODIFIED Requirements

### 需求:别名匹配归档
系统在归档文件时，必须先通过别名查找表将文件的 username 转换为主名称，再执行目标文件夹匹配。移动后的文件必须保留原始 username，禁止改写文件名。

文件名解析时，系统必须使用第一个长度 >= 16 位的纯 ASCII 数字 token 作为 Tweet ID，该 token 之前的所有 token 拼接为 username。系统禁止将空串或短数字识别为 Tweet ID。

#### 场景:别名用户的文件归档
- **当** 文件名解析出 username 为 `alice_art`，且别名表中 `alice_art` 映射到 `alice`，且目标目录存在 `alice` 文件夹
- **那么** 系统将文件移动到 `alice` 文件夹，文件名保持不变（仍含 `alice_art`）

#### 场景:无别名的文件归档
- **当** 文件名解析出 username 不在别名表中
- **那么** 系统使用原始 username 执行现有匹配逻辑，行为不变

#### 场景:别名映射但目标文件夹不存在
- **当** 文件名解析出 username 为 `alice_art`，别名表中映射到 `alice`，但目标目录不存在 `alice` 文件夹
- **那么** 系统跳过该文件并输出提示信息，行为与当前未匹配时一致

#### 场景:CDN 文件名含短数字
- **当** 文件名为 `Winsshiniye888_2021584971287195730_8_kFpUGwReil7Vzo.mp4`
- **那么** 系统必须识别 `2021584971287195730` 为 Tweet ID，`Winsshiniye888` 为 username，忽略 CDN 部分的 `8`

#### 场景:文件名尾部有下划线
- **当** 文件名为 `Jaco242579_1993218560311935011_yFyGyASmM4xO2v4_.mp4`
- **那么** 系统必须识别 `1993218560311935011` 为 Tweet ID，`Jaco242579` 为 username，忽略尾部下划线产生的空串

#### 场景:纯数字 username
- **当** 文件名为 `123456789012345_1993218560311935011_originalname.mp4`（username 为 15 位纯数字）
- **那么** 系统必须识别 `1993218560311935011`（19 位）为 Tweet ID，`123456789012345`（15 位）为 username

#### 场景:无合法 Tweet ID
- **当** 文件名中不包含任何 >= 16 位的纯数字 token
- **那么** 系统必须报告解析失败，跳过该文件
