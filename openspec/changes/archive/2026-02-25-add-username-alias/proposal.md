## 为什么

当前文件归档功能基于 X username 与目标文件夹名进行一对一匹配。但现实中同一个人往往拥有多个 X 账号（如主号、小号、创作号），导致归档时只有主账号的文件能自动归入对应文件夹，其余账号的文件全部被跳过，需要大量手动操作。

## 变更内容

- 新增用户名别名映射机制：支持通过 `{DOWNLOAD_DIR}/username_aliases.txt` 文件定义多个 username 到同一目标文件夹的映射关系
- 映射文件格式：每行一组，逗号分隔，首个 username 为主名称（用于匹配目标文件夹）
- 映射文件不存在时静默忽略，保持现有行为完全不变
- 移动后的文件保留原始 username，不改写文件名

## 功能 (Capabilities)

### 新增功能
- `username-alias`: 用户名别名映射，允许将多个 X username 关联到同一个目标文件夹

## 影响

- 受影响代码：`src/organize_files.rs`（核心变更，增加别名加载与查询逻辑）
- 向后兼容：完全兼容，映射文件不存在时行为与当前完全一致
- 不涉及 X API 交互变更
- 不涉及配置项（.env）变更，采用约定路径 `{DOWNLOAD_DIR}/username_aliases.txt`
