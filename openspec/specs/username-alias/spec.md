### 需求:别名文件加载
系统必须在执行文件归档时，尝试从 `{DOWNLOAD_DIR}/username_aliases.txt` 加载用户名别名映射。映射文件不存在时，系统必须静默忽略并继续使用现有归档逻辑。

#### 场景:别名文件存在
- **当** `{DOWNLOAD_DIR}/username_aliases.txt` 文件存在且内容合法
- **那么** 系统加载所有别名映射关系，构建 alias → primary 的查找表

#### 场景:别名文件不存在
- **当** `{DOWNLOAD_DIR}/username_aliases.txt` 文件不存在
- **那么** 系统静默跳过别名加载，归档行为与未引入此功能前完全一致

### 需求:别名文件格式
别名文件必须支持以下格式：每行一组映射，逗号分隔，首个 username 为主名称。系统必须忽略空行和 `#` 开头的注释行，并对每个 username 去除首尾空格。

#### 场景:标准格式解析
- **当** 文件包含行 `alice, alice_art, alice_photo`
- **那么** 系统将 `alice_art` 和 `alice_photo` 映射到主名称 `alice`

#### 场景:注释和空行
- **当** 文件包含以 `#` 开头的行或空行
- **那么** 系统忽略这些行，不产生错误

#### 场景:仅含单个 username 的行
- **当** 文件包含只有一个 username 的行（如 `alice`）
- **那么** 系统静默跳过该行，不产生映射也不报错

### 需求:别名匹配归档
系统在归档文件时，必须先通过别名查找表将文件的 username 转换为主名称，再执行目标文件夹匹配。移动后的文件必须保留原始 username，禁止改写文件名。

#### 场景:别名用户的文件归档
- **当** 文件名解析出 username 为 `alice_art`，且别名表中 `alice_art` 映射到 `alice`，且目标目录存在 `alice` 文件夹
- **那么** 系统将文件移动到 `alice` 文件夹，文件名保持不变（仍含 `alice_art`）

#### 场景:无别名的文件归档
- **当** 文件名解析出 username 不在别名表中
- **那么** 系统使用原始 username 执行现有匹配逻辑，行为不变

#### 场景:别名映射但目标文件夹不存在
- **当** 文件名解析出 username 为 `alice_art`，别名表中映射到 `alice`，但目标目录不存在 `alice` 文件夹
- **那么** 系统跳过该文件并输出提示信息，行为与当前未匹配时一致
